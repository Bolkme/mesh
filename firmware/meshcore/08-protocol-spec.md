本文档整理了 MeshCore 的核心协议规范，基于官方文档和源码分析。

## 1. 数据包格式 (Packet Format)

**官方文档**: `docs/packet_format.md`

### V1 数据包结构

```
┌─────────┬──────────────┬─────────────┬──────────┬─────────┐
│ header  │ trans_codes  │ path_length │   path   │ payload │
│ 1 byte  │ 4 bytes(可选) │  1 byte     │ ≤64 bytes│ ≤184 bytes│
└─────────┴──────────────┴─────────────┴──────────┴─────────┘
```

### Header 字节详解

```
位：7 6 5 4 3 2 1 0
    │ │ └───┴───┴───┴──┐
    │ │      载荷类型   │
    │ │    (4 位)       │
    │ └───┘ └─┴─┘
    │      版本  路由类型
    │     (2 位)  (2 位)
```

**位掩码**:
- `0x03` (位 0-1): 路由类型
- `0x3C` (位 2-5): 载荷类型
- `0xC0` (位 6-7): 载荷版本

### 路由类型

| 值 | 名称 | 描述 | 使用场景 |
|----|------|------|----------|
| `0x00` | TRANSPORT_FLOOD | 泛洪 + 传输代码 | 区域广播 |
| `0x01` | FLOOD | 泛洪路由 | 普通广播 |
| `0x02` | DIRECT | 直接路由 | 点对点 |
| `0x03` | TRANSPORT_DIRECT | 直接 + 传输代码 | 区域点对点 |

### 载荷类型

| 值 | 名称 | 描述 | 加密 |
|----|------|------|------|
| `0x00` | REQ | 请求 | ✅ |
| `0x01` | RESPONSE | 响应 | ✅ |
| `0x02` | TXT_MSG | 文本消息 | ✅ |
| `0x03` | ACK | 确认 | ❌ |
| `0x04` | ADVERT | 节点广告 | ❌ |
| `0x07` | ANON_REQ | 匿名请求 | ✅ |
| `0x08` | PATH | 返回路径 | ✅ |
| `0x09` | TRACE | 路径追踪 | ❌ |
| `0x0A` | MULTIPART | 多部分包 | ✅ |
| `0x0B` | CONTROL | 控制数据 | ❌ |
| `0x0F` | RAW_CUSTOM | 自定义 | 自定义 |

## 2. 载荷格式 (Payloads)

**官方文档**: `docs/payloads.md`

### 2.1 节点广告 (ADVERT)

**用途**: 宣告节点存在，广播节点信息

```
┌─────────────┬───────────┬───────────┬──────────────┐
│ public_key  │ timestamp │ signature │   appdata    │
│  32 bytes   │ 4 bytes   │ 64 bytes  │  可变长度    │
└─────────────┴───────────┴───────────┴──────────────┘
```

**Appdata 结构**:
```
┌───────┬──────────┬───────────┬──────────┬──────────┬────────┐
│ flags │ latitude │ longitude │ feature1 │ feature2 │  name  │
│1 byte │4 bytes*  │ 4 bytes*  │ 2 bytes* │ 2 bytes* │ 可变   │
└───────┴──────────┴───────────┴──────────┴──────────┴────────┘
* 可选字段，由 flags 决定
```

**Flags 定义**:
| 位 | 值 | 含义 |
|----|-----|------|
| 0 | `0x01` | 聊天节点 |
| 1 | `0x02` | 中继器 |
| 2 | `0x03` | 房间服务器 |
| 3 | `0x04` | 传感器 |
| 4 | `0x10` | 包含位置 |
| 7 | `0x80` | 包含名称 |

### 2.2 确认包 (ACK)

**用途**: 确认消息已接收

```
┌──────────┐
│ checksum │
│ 4 bytes  │
└──────────┘
```

**Checksum**: CRC 校验和（消息时间戳 + 文本 + 发送者公钥）

### 2.3 数据包 (REQ/RESPONSE/TXT_MSG)

**通用结构**:
```
┌──────────────┬────────────┬───────────┬────────────┐
│ dest_hash    │ src_hash   │ cipher_mac│ ciphertext │
│ 1 byte       │ 1 byte     │ 2 bytes   │ 可变长度   │
└──────────────┴────────────┴───────────┴────────────┘
```

**加密**: 使用 ECDH 共享密钥加密

### 2.4 文本消息 (TXT_MSG)

**解密后结构**:
```
┌───────────┬──────────────────┬─────────────┐
│ timestamp │ txt_type+attempt │   message   │
│ 4 bytes   │ 1 byte           │  可变长度   │
└───────────┴──────────────────┴─────────────┘
```

**Txt_Type** (高 6 位):
| 值 | 类型 | 描述 |
|----|------|------|
| `0x00` | 纯文本 | 普通消息 |
| `0x01` | CLI 命令 | 命令行 |
| `0x02` | 签名文本 | 带签名的消息 |

**Attempt** (低 2 位): 重传次数 (0-3)

### 2.5 返回路径 (PATH)

**结构**:
```
┌─────────────┬──────────┬────────────┬───────────┐
│ path_length │   path   │ extra_type │   extra   │
│  1 byte     │ N bytes  │  1 byte    │ 可变长度  │
└─────────────┴──────────┴────────────┴───────────┘
```

**用途**: 返回消息经过的路径，用于路由学习

## 3. 伴随协议 (Companion Protocol)

**官方文档**: `docs/companion_protocol.md`

### BLE 连接

**服务 UUID**: `6E400001-B5A3-F393-E0A9-E50E24DCCA9E`

**特征**:
| 方向 | UUID | 用途 |
|------|------|------|
| App→固件 | `6E400002-B5A3-F393-E0A9-E50E24DCCA9E` | RX (写入) |
| 固件→App | `6E400003-B5A3-F393-E0A9-E50E24DCCA9E` | TX (通知) |

### 连接流程

```
1. 扫描设备
   ↓
2. 连接 GATT
   ↓
3. 发现服务和特征
   ↓
4. 启用 TX 通知
   ↓
5. 发送 CMD_APP_START
   ↓
6. 发送 CMD_DEVICE_QUERY
   ↓
7. 获取频道和联系人
   ↓
8. 正常通信
```

### 关键命令

#### CMD_APP_START (初始化)

```
命令：01 03 6d 63 63 6c 69 00 00 00 00
      │  │  └────────"mccli"────────┘
      │  └─ 长度
      └──── 命令码
```

**响应**: PACKET_OK (`0x00`)

#### CMD_GET_CHANNEL (获取频道)

```
命令：1F <channel_index>
      │   └────────── 0-7
      └────── 命令码
```

**响应**: PACKET_CHANNEL_INFO (`0x12`)

#### CMD_SET_CHANNEL (设置频道)

```
命令：20 <idx> <name:32bytes> <secret:32bytes>
      │   │    │                └── 私密频道密钥
      │   │    └─── 频道名 (空填充)
      │   └── 频道索引 (0=公共，1-7=私密)
      └────── 命令码
```

**总长度**: 66 字节

#### CMD_SEND_CHANNEL_MSG (发送频道消息)

```
命令：03 00 <idx> <timestamp:4bytes> <message>
      │  │   │    │                  └── UTF-8 消息
      │  │   │    └── Unix 时间戳 (小端)
      │  │   └── 频道索引
      │  └── 保留
      └───── 命令码
```

**响应**: PACKET_MSG_SENT (`0x06`)

### 响应类型

| 代码 | 名称 | 描述 |
|------|------|------|
| `0x00` | PACKET_OK | 成功 |
| `0x01` | PACKET_ERROR | 错误 |
| `0x06` | PACKET_MSG_SENT | 消息已发送 |
| `0x07` | PACKET_CONTACT_MSG_RECV | 收到联系人消息 |
| `0x08` | PACKET_CHANNEL_MSG_RECV | 收到频道消息 |
| `0x0D` | PACKET_DEVICE_INFO | 设备信息 |
| `0x12` | PACKET_CHANNEL_INFO | 频道信息 |

## 4. KISS 调制解调器协议

**官方文档**: `docs/kiss_modem_protocol.md`

### 帧格式

```
┌──────┬────────┬──────────┬──────┐
│ FEND │ CMD    │ DATA     │ FEND │
│0xC0  │1 byte  │可变长度  │0xC0  │
└──────┴────────┴──────────┴──────┘
```

### KISS 命令

| 命令 | 值 | 描述 |
|------|-----|------|
| Data | `0x00` | 数据包 |
| Init | `0x21` | 初始化 |

### 使用场景

- 主机应用通过串口与 MeshCore 通信
- AX.25 TNC 兼容
- 适合集成到现有系统

## 6. 安全机制

### 加密流程

```
1. 节点 A 和 B 各有 Ed25519 密钥对
2. 交换公钥 (通过 ADVERT 包)
3. 计算 ECDH 共享密钥
4. 使用共享密钥加密数据
5. 附加 2 字节 MAC
```

### 密钥管理

- **公钥**: 32 字节 Ed25519 公钥
- **私钥**: 32 字节，本地保存
- **共享密钥**: ECDH 计算得出
- **节点哈希**: 公钥首字节 (用于快速识别)

### 访问控制 (ACL)

**传感器节点支持**:
- 白名单机制
- 仅允许授权节点访问
- 通过 CLI 命令配置

## 7. 路由算法

### 泛洪路由 (Flood)

```
发送方：
1. 创建包，设置 ROUTE_TYPE_FLOOD
2. 广播到空中
3. 等待重传时机

中继方：
1. 接收包
2. 检查包哈希是否在 seen 表中
3. 已见过 → 丢弃
4. 未见过 → 添加到 seen 表，重传
```

### 直接路由 (Direct)

```
发送方：
1. 查找目标节点的路径
2. 创建包，设置 ROUTE_TYPE_DIRECT
3. 填充 path[] 字段
4. 发送

中继方：
1. 接收包
2. 检查 path[0] 是否是自己的哈希
3. 是 → 移除首字节，转发下一跳
4. 否 → 丢弃
```

### 路径发现

**机制**:
1. 发送 REQ 包到目标
2. 目标回复 RESPONSE，附带返回路径
3. 学习路径并存储

## 8. V2 协议规划

### 改进方向

1. **更大的哈希**: 2 字节 (当前 1 字节)
   - 减少冲突概率
   - 支持更多节点

2. **更大的 MAC**: 4 字节 (当前 2 字节)
   - 增强安全性
   - 减少伪造可能

3. **路径哈希**: 压缩路径表示
   - 减少包大小
   - 提高效率

4. **动态编码率**: 根据链路质量调整
   - 强链路：低冗余
   - 弱链路：高冗余

## 📊 协议对比

| 特性 | MeshCore V1 | Meshtastic |
|------|-------------|------------|
| 包格式 | 自定义 | 自定义 (基于 Protobuf) |
| 加密 | ECDH + 自定义 | AES-256 |
| 路由 | 泛洪 + 直接 | 泛洪 + 动态 |
| 最大包大小 | ~256 字节 | ~240 字节 |
| 路径长度 | 64 跳 | 7 跳 (默认) |
| 节点哈希 | 1 字节 | 4 字节 |
