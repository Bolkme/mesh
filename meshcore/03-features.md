# MeshCore 功能特性

## ✅ 核心功能

### 1. 多跳数据包路由

```
节点 A ─────► 节点 B ─────► 节点 C ─────► 节点 D
  │            │            │            │
  └────────────┴────────────┴────────────┘
              多跳通信路径
```

- 消息可通过多个中间节点中继
- 扩展单无线电覆盖范围
- 可配置最大跳数
- 防止过度网络流量

### 2. 固定节点角色

| 角色 | 转发消息 | 用途 |
|------|----------|------|
| **Companion** | ❌ 不转发 | 连接用户设备（手机/电脑） |
| **Repeater** | ✅ 转发 | 扩展网络覆盖 |
| **Room Server** | ✅ 转发 + 存储 | BBS 公告板功能 |
| **Sensor** | ✅ 转发 + 采集 | 远程传感器数据 |

**设计优势**: Companion 节点不转发消息，避免不良路由路径被使用。

### 3. 去中心化网络

- 无需中央服务器
- 无需互联网连接
- 网络自愈能力
- 抗单点故障

### 4. 低功耗设计

- 适合电池供电设备
- 适合太阳能供电设备
- 嵌入式系统优化

### 5. 多平台客户端

```
│ Web 浏览器   │ ✅ https://app.meshcore.nz   │

└──────────────┴──────────────────────────────┘
```

## 🔧 配置与管理

### 固件刷写

- **工具**: https://flasher.meshcore.co.uk
- **方式**: 预编译二进制，无需手动编译
- **支持设备**: Heltec、RAK Wireless 等

### 设备配置

- **工具**: https://config.meshcore.dev
- **连接**: USB 串口配置
- **远程管理**: 通过 LoRa 使用移动应用的远程管理功能

## 📡 通信模式

### 1. 点对点聊天

```
用户 A ◄──► Companion A ◄──► Repeater ◄──► Companion B ◄──► 用户 B
```

### 2. 广播/房间消息

```
用户 A ◄──► Room Server ◄──► 所有连接用户
              （BBS 模式）
```

### 3. 传感器数据

```
Sensor Node ◄──► Repeater ◄──► 数据收集端
     │
     └──► 遥测数据 + 告警
```
