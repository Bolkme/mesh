## 📚 核心组件

### 1. MeshCore 库（C++）

- **定位**: 轻量级、可移植的 C++ 库
- **功能**: 实现多跳数据包路由
- **设计原则**:
  - 保持简单，避免高级语言的复杂层次
  - 无动态内存分配（setup/begin 除外）
  - 代码简洁，无不必要的抽象层

### 2. 固件类型

| 固件类型 | 用途 | 连接方式 |
|----------|------|----------|
| **Companion Radio** | 连接外部聊天应用 | BLE、USB、WiFi |
| **KISS Modem** | 串行 KISS 协议桥接 | 串口 |
| **Simple Repeater** | 消息中继 | LoRa |
| **Simple Room Server** | BBS 服务器 | LoRa + USB 配置 |
| **Simple Sensor** | 传感器节点 | LoRa |
| **Simple Secure Chat** | 安全终端通信 | 串口/USB |

## 🔌 硬件支持

MeshCore 支持多种 LoRa 设备：

- **Heltec** 系列
- **RAK Wireless** 系列
- 其他基于 LoRa 的硬件

### 刷写流程

1. 访问 https://flasher.meshcore.co.uk
2. 选择支持的设备
3. 选择固件类型（Companion / Repeater / Room Server）
4. 使用 Adafruit ESPTool 等工具刷写预编译二进制文件
5. 通过串口控制台与网络交互

---

## 🔐 安全特性

- **去中心化**: 无单点故障
- **自愈网络**: 节点失效时自动重新路由
- **安全通信**: 支持加密（V2 协议规划中）
- **ACL 支持**: 传感器节点已有访问控制列表（Repeater/Room Server 规划中）

---

## 📋 协议细节

### KISS Modem 协议

- 文档位置：`/docs/kiss_modem_protocol.md`
- 用于主机应用与 MeshCore 设备的串口通信

### 路由机制

- **可配置跳数**: 平衡网络效率与流量控制
- **固定角色**: Companion 节点不转发消息，避免不良路由路径
- **零跳邻居发现**: 增强型（规划中）
- **手动路径支持**: 往返手动路径（规划中）
