## 📋 执行摘要

Meshtastic 固件是运行在 LoRa 设备上的核心软件，基于 **PlatformIO** 开发环境，支持多种 MCU 平台（ESP32、nRF52、RP2040 等）。固件采用模块化设计，支持 OTA 更新、低功耗管理、加密通信等核心功能。

**本报告涵盖：** 固件架构、开发环境、构建流程、核心功能、配置选项、调试方法。

---

## 一、固件概述

### 1.1 固件定位

Meshtastic 固件是设备的"大脑"，负责：

- 📻 LoRa 无线电通信管理
- 🔗 Mesh 网络协议实现
- 🔐 AES-256 加密/解密
- 📱 蓝牙/WiFi/USB 连接
- 📺 OLED/E-Ink 显示驱动
- 📍 GPS 数据处理
- 🔋 电源管理

### 1.2 技术栈

| 组件 | 技术 |
|------|------|
| **开发语言** | C++ |
| **开发环境** | PlatformIO |
| **支持平台** | ESP32, nRF52, RP2040, STM32 |
| **无线电库** | RadioLib (SX126x/SX127x/LR11xx) |
| **加密库** | AES-256 |
| **文件系统** | LittleFS, SPIFFS |
| **通信协议** | 蓝牙 5.0, WiFi, USB Serial |

## 三、开发环境

### 3.1 PlatformIO 简介

**PlatformIO** 是一个跨平台的嵌入式开发环境：

- ✅ 支持 1000+ 开发板
- ✅ 统一的构建系统
- ✅ 库依赖管理
- ✅ 调试支持
- ✅ CI/CD 集成

### 3.2 环境搭建

#### 方式 1：VSCode + PlatformIO 插件（推荐）

```bash
# 1. 安装 VSCode
# 2. 安装 PlatformIO IDE 扩展
# 3. 克隆 Meshtastic 仓库
git clone https://github.com/meshtastic/firmware.git
cd firmware

# 4. PlatformIO 自动识别项目
# 5. 选择目标板子，点击构建
```

#### 方式 2：命令行

```bash
# 安装 PlatformIO Core
pip install platformio

# 克隆项目
git clone https://github.com/meshtastic/firmware.git
cd firmware

# 构建（以 RAK4631 为例）
pio run -e rak4631

# 烧录
pio run -e rak4631 -t upload
```

## 四、核心功能详解

### 4.1 LoRa 通信

#### 无线电支持

| 芯片系列 | 型号 | 支持状态 |
|----------|------|----------|
| **SX126x** | SX1262, SX1268 | ✅ 推荐 |
| **LR11xx** | LR1110, LR1121 | ✅ 推荐 |
| **SX127x** | SX1276, SX1278 | ⚠️ 老款 |
| **SX128x** | SX1280 | ✅ 2.4GHz |

#### 关键参数配置

```cpp
// 频率设置（根据地区）
float frequency = 915.0;  // 美国

// 带宽
float bandwidth = 125.0;  // 125kHz

// 扩频因子
uint8_t spreadingFactor = 10;  // SF7-SF12

// 编码率
uint8_t codingRate = 5;  // 4/5, 4/6, 4/7, 4/8

// 发射功率
int8_t power = 22;  // dBm
```

### 4.2 Mesh 路由协议

#### 消息转发机制

```
1. 节点 A 发送消息
       ↓
2. 邻近节点接收并存储
       ↓
3. 检查是否已转发过（去重）
       ↓
4. 根据 TTL 决定是否转发
       ↓
5. 转发到下一跳
```

#### 路由优化

- **SNR 感知：** 优先选择信号质量好的路径
- **跳数限制：** 默认最大 7 跳
- **广播抑制：** 避免消息风暴
- **ACK 机制：** 可靠传输（可选）

### 4.3 加密系统

#### AES-256 加密

```cpp
// 频道密钥（32 字节）
uint8_t channelKey[32] = {
    0x00, 0x01, 0x02, ..., 0x1F
};

// 加密过程
1. 生成随机 Nonce
2. 使用 AES-256 加密消息
3. 附加 Nonce 到密文
4. 发送
```

#### 安全特性

- ✅ 每个频道独立密钥
- ✅ 默认频道使用公开密钥
- ✅ 私有频道可自定义密钥
- ✅ 消息完整性校验

### 4.4 电源管理

#### 低功耗模式（nRF52）

| 模式 | 电流 | 说明 |
|------|------|------|
| **Active** | ~5mA | 正常运行 |
| **Sleep** | ~30μA | 等待消息 |
| **Deep Sleep** | ~5μA | 长时间待机 |

#### 电源状态机 (PowerFSM)

```
┌─────────┐    超时    ┌─────────┐
│  Active  │ ────────► │  Sleep   │
└─────────┘           └─────────┘
     ▲                     │
     │  消息/事件          │  超时
     │                     ▼
     │              ┌─────────────┐
     └──────────────│ Deep Sleep  │
                    └─────────────┘
```

### 4.5 显示支持

#### 支持的显示屏

| 类型 | 接口 | 支持设备 |
|------|------|----------|
| **OLED** | I2C | 0.96" 128x64 |
| **E-Ink** | SPI | 2.13" 212x104 |
| **TFT** | SPI | 各种尺寸 |

#### 显示内容

- 当前频道
- 信号强度 (RSSI/SNR)
- 电池电量
- GPS 坐标
- 消息列表
- 网络拓扑

### 4.6 GPS 集成

#### 支持的 GPS 模块

- ✅ 内置 GPS（T-Beam, T-Echo）
- ✅ 外部 GPS（UART 连接）
- ✅ 软件 GPS（模拟位置）

#### GPS 功能

- 位置共享
- 轨迹记录
- 时间同步
- 高度/速度数据

## 五、固件构建

### 5.1 构建流程

```bash
# 1. 克隆仓库
git clone https://github.com/meshtastic/firmware.git
cd firmware

# 2. 选择分支（稳定版或开发版）
git checkout master          # 稳定版
git checkout alpha           # 测试版

# 3. 构建目标板子
pio run -e <环境名>

# 4. 生成固件文件
# 输出：.pio/build/<环境名>/firmware.bin
```

### 5.2 常见构建环境

| 环境名 | 目标板子 | 命令 |
|--------|----------|------|
| `rak4631` | RAK4631 | `pio run -e rak4631` |
| `tbeam` | LILYGO T-Beam | `pio run -e tbeam` |
| `heltec_v3` | Heltec V3 | `pio run -e heltec_v3` |
| `seeed_xiao` | Seeed Xiao | `pio run -e seeed_xiao` |
| `native` | Linux 原生 | `pio run -e native` |

### 5.3 自定义编译选项

```ini
; platformio.ini 中添加
build_flags =
    -D USER_CONFIG_VERSION=1
    -D MESHTASTIC_EXCLUDE_GPS=1      ; 禁用 GPS
    -D MESHTASTIC_EXCLUDE_WIFI=1     ; 禁用 WiFi
    -D MESHTASTIC_EXCLUDE_BLUETOOTH=1 ; 禁用蓝牙
    -D PIN_LED1=12                   ; 自定义 LED 引脚
```

## 六、固件烧录

### 6.1 烧录方式

#### 方式 1：Web Flasher（推荐新手）

1. 访问 https://flasher.meshtastic.org
2. 选择设备型号
3. 选择固件版本
4. 点击 "Flash"
5. 等待完成

#### 方式 2：命令行烧录

```bash
# ESP32 设备
esptool.py --chip esp32 write_flash -z 0x1000 firmware.bin

# nRF52 设备（使用 nrfutil）
nrfutil dfu usb-serial -pkg firmware.zip -p /dev/ttyACM0
```

#### 方式 3：PlatformIO 烧录

```bash
pio run -e rak4631 -t upload
```

### 6.2 DFU 模式

**nRF52 设备进入 DFU 模式：**

1. 双击复位按钮
2. 指示灯变为呼吸灯
3. 设备显示为 USB 设备
4. 进行烧录

### 8.1 串口日志


#### 日志级别

| 级别 | 说明 | 输出量 |
|------|------|--------|
| **DEBUG** | 调试信息 | 最多 |
| **INFO** | 一般信息 | 适中 |
| **WARN** | 警告 | 较少 |
| **ERROR** | 错误 | 最少 |

### 8.2 常见问题排查

#### 问题 1：设备无法启动

```
可能原因：
- 电源问题
- 固件损坏
- 引脚配置错误

排查步骤：
1. 检查电源电压
2. 重新烧录固件
3. 查看串口日志
```

#### 问题 2：LoRa 无法通信

```
可能原因：
- 频率设置错误
- 天线未连接
- 区域配置不匹配

排查步骤：
1. 确认区域设置正确
2. 检查天线连接
3. 使用频谱仪检测
```

#### 问题 3：蓝牙无法连接

```
可能原因：
- 蓝牙未启用
- PIN 码错误
- 距离太远

排查步骤：
1. 确认蓝牙已启用
2. 重置 PIN 码
3. 靠近设备重试
```
